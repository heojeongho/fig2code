# 단일 프롬프트 버전 (Claude Code 스킬을 못 쓰는 환경용)

아래 전체를 복사해 피그마 URL만 바꿔서 사용하세요. 스킬 버전과 달리 검증 스크립트를
매번 에이전트가 새로 작성하게 되므로, 가능하면 스킬 버전을 권장합니다.

---

피그마 디자인을 픽셀 퍼펙트 반응형 HTML로 퍼블리싱해줘: {피그마 URL}

요구사항:

1. **구조**: get_metadata로 PC/모바일 프레임과 섹션 구조를 파악하고, get_screenshot으로 두 프레임의 원본 해상도 풀샷을 저장해 비교 기준으로 삼아줘. 섹션 단위로 get_design_context를 호출해 좌표·폰트·색상을 추출하고, 응답이 너무 크면 서브노드로 쪼개서 받아.

2. **에셋**: 텍스트·카드·버튼은 실제 HTML/CSS로 구현하고, 일러스트·사진·손글씨·로고만 에셋으로 추출해. 주의사항:
   - download_assets SVG에는 페이지 배경 rect와 부모 프레임 흰 rect가 박혀 나오니 제거하되, `<defs>`(clipPath) 안의 rect는 절대 건드리지 마.
   - PNG export는 흰 배경이 박히니, 투명이 필요한 이미지는 rawImages URL(원본, RGBA)을 받아 CSS crop으로 재현해.
   - 같은 모양도 PC/모바일 노드의 회전·비율이 다를 수 있으니 프레임별로 각각 export해.

3. **반응형(분기형, 스케일 금지)**: ≥1024px는 PC 레이아웃 — 요소 좌표를 `calc(50% ± Npx)` 중앙 앵커로 변환해 콘텐츠는 중앙 정렬, 배경(히어로 스카이라인 등)은 `object-fit: cover`로 풀블리드. <1024px는 모바일 레이아웃 — 플로우 스택 + 중앙 앵커 + 유동 카드 폭, 320px까지 미세 미디어쿼리로 대응. 모바일 프레임의 iOS 상태바는 제외하고 y를 시프트해. 중앙정렬 텍스트의 꼬리 공백은 `&nbsp;`로 치환해(collapse로 인한 시프트 방지).

4. **인터랙션**: 아코디언(하나만 열림, 시안의 열림 상태를 기본값으로), 토글, 드롭다운/햄버거 메뉴, 무한 마퀴 캐러셀(t=0 위치가 시안과 일치, prefers-reduced-motion 시 정지), 앵커 스무스 스크롤을 구현해.

5. **검증 루프(랄프루프)**: 로컬 서버 + playwright(`--hide-scrollbars`, reducedMotion 에뮬레이션)로 디자인 기준 폭 풀페이지를 캡처하고, 시안 풀샷과 2×8 분면으로 픽셀 diff를 산출해. 상위 분면은 시안|웹|히트맵 3중 패널로 저장해 원인을 파악하고, 의심 요소는 PIL로 bbox를 실측해 수치로 보정해. 텍스트 안티앨리어싱 노이즈와 구조 문제를 구분하려면 2px 가우시안 블러 후 diff를 봐. 블러 diff가 1%대에 들고 요소 bbox 오차가 2px 이하가 될 때까지 반복해. 마지막으로 320~2560 폭 스윕(가로 스크롤 0 확인)과 인터랙션 자동 테스트를 돌려줘.

6. **보고**: 최종 diff 수치, 남은 차이의 원인(폰트 래스터라이징 등 제거 불가 항목), 시안에 없어 placeholder로 채운 콘텐츠 목록을 정리해줘.
